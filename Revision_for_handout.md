# Minimax

算法介绍可以见比如 (oiwiki)[[Alpha–Beta 剪枝 - OI Wiki](https://oi-wiki.org/search/alpha-beta/)] 。

简言之，我们对每一个局面设定一个值，指代这个局面谁占优势。理想情况下在这个信息透明没有随机性的游戏中这个值是$-\infty$（先手必败）/$0$（平局）/$+\infty$（先手必胜），但是当然实际上我们计算不出这个值，所以除了胜负已分的局面这个权值一般是由一个估值函数计算的估值。

搜索的过程就是，依次向下搜索每一步直到达到深度限制，并对叶子节点估算其权值。之后对于非叶子节点，如果当前节点对应下一步先手行动则令其权值为其子节点的最大值（并选定这个节点的决策为走向对应子节点的决策），否则（当前下一步为后手行动）为最小值。

估值函数在这里发挥很大的作用，在五子棋这种游戏中常见的简单估值函数可能是计算活2、活3、跳3、冲4等等特定旗形的数量，当然棋形的位置（中间还是边角）也可以纳入考虑（毕竟在边角走棋显然是不太优的）

更详细的介绍和可能需要用到的更多优化在 handout.pdf 中给的 repo 链接中给出。

值得注意的一点细节是，在 Minimax 部分你需要支持五子棋 3 步交换的规则（于是你如果是先手，需要期待第3步后的局面尽量平衡），而 AlphaZero 部分为了简化不要求支持。

这个部分你可以尽情发挥人类智慧，希望你能在其中找到一点乐趣。

# AlphaZero

关于深度学习、强化学习的部分在 handout.pdf 中有较为详细的说明。

鉴于各位在大一的课程中可能没有接触过机器学习，而我们的小学期的演讲作业也不再是去年的~~变态的~~阅读大量LLM/多模态模型相关的论文来做报告，我在思考有没有必要关于这个部分的基础知识做一些适当的讲解（我想大概是需要的）。去年的情况是每一个项目组有一间单独的教室，并且五子棋组事实上上了大概2个课时的机器学习相关的入门内容。（其他部分组是直接分发材料自行阅读的）然而本学期目前并没有讲课的安排。后续如果有需要，我可能会发额外的资料。

回归正题，讲一下各位需要做什么（基础知识见 handout.pdf，里面的内容较为详尽了），我给各位提供2个选项：

## Option 1.（去年的要求）

首先补全 `mcts_pure.py` 来熟悉一下基本 mcts 的结构，然后参照 handout.pdf 补全 `policy_value_net.py` 和 `mcts_alpha_zero.py` 来分别完成估值网络的搭建和 AlphaZero 的结构。之后（1）导入我们已经提供给你的 11*11 的五子棋模型（model/11_11_5.pt），你就可以看到你的模型有不错的棋力。（2）修改 train.py 等处的参数训练一个 3\*3 的井字棋ai，这个预期非常快。

这个部分的评价标准主要是你的代码实现的正确性（因为有 vibe coding 的存在，这部分分数在现在应当是基本白给的），以及在 report 中你展示的理解。实际上没有太多的工作量，评价理解的部分也很具主观性，所以我试图做出一点变革。

## Option 2.

同样如 Option 1. 补全所需的3个部分文件，但是后续改成训练一个比如 5\*5 棋盘的四子棋（这里的四子棋不是指你去搜索引擎可以找到的棋子会“掉落”的四子棋，仅指五子棋的只需连成4子的推广版本；这里的参数 $(5,5,4)$ 仅供参考，你可以根据实际训练速度进行调整，这个参数推荐的基础只是在这个局面双方较为均衡——虽然似乎先手必胜但是没有井字棋那么容易胜）。你可以研究一下采用不同的 resblock 的数量、不同的学习的参数和方式，抑或是尝试修改网络的结构，对训练过程、棋力的影响。你的目的可以是提高棋力、提高训练效率、或者单纯探寻一下不同因素是如何发挥作用的。 总之我想这个方向是比较开放性的，如果你能提出很有创新型的 idea 我会额外给创新性加分（因为我设计的总分大于100%，你也可以理解为这部分分数可以溢出到 Minimax 部分）。你的研究成果应当呈现在 report 中。
